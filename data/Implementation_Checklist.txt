□ Step 16: Create Customer List Page
  
  File: pages/Customers.jsx
  
  Display:
    ├─ List of all customers
    ├─ Customer name (link to detail)
    ├─ Total MRR (sum of all their deals)
    ├─ Number of deals
    ├─ Website link
    └─ Edit/Delete buttons
  
  Query:
    ```jsx
    const { data: customers } = await supabase
      .from('customers')
      .select('*, deals(monthly_cost)')
    ```

    │  ├─ Contract

    │  └─ Latest invoice

    └─ Edit/Renew buttons

□ Step 19: Create Partner View
  
  File: pages/Partner.jsx

    ├─ Partner info (name, type, website)
    ├─ Services offered
    ├─ List of customers using this partner
    ├─ Total revenue from this partner
    ├─ Support URL

□ Step 20: Add Renewal Alerts Feature
  
  File: components/RenewalAlerts.jsx
  
  Display:
    ├─ Deals renewing in next 90 days
    ├─ Color-coded by urgency (red < 30, yellow < 60, green < 90)
    ├─ Click to see details
    └─ Quick action to contact customer
  
  Query:
    ```jsx
    const { data: upcoming } = await supabase
      .from('deals')
      .select('*')
      .where('renewal_notice_date', 'lt', '90_days_from_now')
      .where('status', 'eq', 'ACTIVE')
    ```

STUCK? HERE'S THE QUICK FIX
================================================================================

"Imports are not showing up in Supabase"
  → Check: .env.local has correct SUPABASE_URL and SUPABASE_KEY
  → Check: Supabase → Settings → API policies allow inserts
  → Check: Browser console for error messages

"Files won't upload"
  → Check: candid_documents bucket is PRIVATE not PUBLIC
  → Check: File storage is enabled in Supabase
  → Check: File size < 500MB

"Customer list is empty"
  → Check: Did you run "Import Customers" button?
  → Check: Supabase → customers table has rows
  → Check: Customer query in your page component

"Renewal dates look wrong"
  → Check: contract_end_date in JSON is correct format (YYYY-MM-DD)
  → Check: renewal_notice_date = 90 days before end date
  → Check: Database query is filtering correctly
